# CallCatch architecture

## Components

1. Clinic handset (Android + MacroDroid). Owns the clinic's real phone number. On a missed call it fires two actions within seconds: an HTTP POST to the backend with the caller number, and an SMS back to the caller with a Telegram deep link. The SMS comes from the clinic's own SIM, so the caller sees the number they just dialed.
2. Convex backend. Source of truth. Tables: clinics, leads, events, slots, waitlist, payments. An HTTP action receives the missed-call webhook, normalizes the caller number to its last 10 digits, deduplicates repeat calls, and appends to an append-only event timeline per lead. Crons handle re-nudge eligibility, tentative-slot expiry, and cold-marking.
3. Hermes gateway + clinic skill. The Telegram bot is served by a running Hermes gateway. The skill resolves `/start <caller-digits>` deep links to the lead created by the webhook, links the Telegram user to the lead, then handles the conversation: answers from the clinic brief (services, price bands, hours), holds tentative slots, and logs every turn back to Convex. Hermes session memory keeps per-user context across turns and repeat visits.
4. Voice notes. Longer answers are synthesized with ElevenLabs (mp3), converted with ffmpeg to mono OGG/Opus, and sent as Telegram voice messages.
5. Lead board. Vite + React on Cloudflare Pages, reading Convex live queries: leads with stage, response time (missed call to first bot reply), held slots, revenue at stake, and the full per-lead event timeline.
6. Payments. Dodo Payments checkout link for the clinic pilot subscription; payment webhook mirrors into the payments table.

## Data flow: missed call to held slot

```text
MacroDroid                Convex                     Hermes (Telegram)
    |  POST /missed-call     |                             |
    |----------------------->|  create lead + event        |
    |  SMS deep link         |                             |
    |  (to caller)           |                             |
                             |   /start <digits>           |
                             |<----------------------------|  match lead by digits
                             |  linkTelegram, logTurn      |  greet with context
                             |<--------------------------->|  FAQ answers (clinic brief)
                             |  holdSlot + event           |  offer + hold slot
                             |<----------------------------|  voice note (ElevenLabs)
        dashboard reads live queries: stage, timeline, response time
```

## Design decisions

- Phone identity is the last 10 digits (Indian mobile format), stored as a string. All matching (webhook dedupe, deep-link resolution) uses this key.
- The event table is append-only; lead.stage is a denormalized convenience that always has a corresponding event. The demo proof minute scrolls this timeline.
- Telegram before WhatsApp: WhatsApp Business Cloud API business-initiated messages require template approval and, on a sandbox number, cap verified recipients at five. Telegram makes day-one conversations real for anyone. The channel layer is Hermes's gateway, so the WhatsApp Cloud adapter (already present in Hermes) swaps in once the clinic's number is registered. The first touch stays SMS in both worlds.
- Crons never send messages directly; they mark nudge eligibility as events. The send path is owned by the skill/gateway layer, keeping one code path for all outbound messages.
- No seeded or fabricated data. Demo numbers are generated by real calls made during the event.

## Failure modes

- Caller withholds number: no digits, no deep link match; the skill asks which number they called from and matches manually.
- Deep link opened by a different person than the caller: link binds on first /start; subsequent conflicting /start creates a fresh unmatched conversation flagged for the front desk.
- MacroDroid killed by battery optimizer: exempted at setup; the clinic line phone stays on charge.
- Never-started leads (no Telegram): single SMS re-nudge queued for human send, surfaced on the dashboard.

## Roadmap

- WhatsApp Business Cloud API on the clinic's registered number (adapter already in Hermes).
- Exotel missed-call webhook for multi-line and landline clinics.
- Front-desk slot confirmation flow and calendar integration.
- Outbound voice calls via conversational TTS for high-value leads.
